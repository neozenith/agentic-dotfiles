# G1: Cache the widget lookup path

> - **Index:** [README.md](./README.md)
> - **Architecture:** [G1 increment](./DISCOVERY.md#g1-increment)
> - **Depends on:** none
> - **Blocks:** none

Repeated widget lookups are served from an in-memory LRU cache through the existing public
`lookup_widget` interface.

## Context

`src/widgets.py::lookup_widget` recomputes the derived display view on every call; there is no
caching seam anywhere in the lookup path.

## Outputs

| File | Change |
|------|--------|
| `src/widgets.py` (python) | `lookup_widget` becomes cache-backed (stdlib LRU), same public signature |
| `tests/test_cache.py` (proof-of-execution) | asserts `cache_info().hits >= 1` after two identical lookups through the real interface |

## ADR1.1: Use functools.lru_cache on the pure lookup seam

- **Decision:** decorate the pure lookup seam with `functools.lru_cache`.
- **Why:** stdlib, zero dependencies, and `cache_info()` gives a falsifiable hit counter.
- **Rejected:** hand-rolled dict cache (reinvents eviction); external cache service (absurd scale).
