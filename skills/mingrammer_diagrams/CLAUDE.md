# CLAUDE.md — maintaining the `mingrammer_diagrams` skill

Guidance for an assistant editing files under `.claude/skills/mingrammer_diagrams/`. Read the
ADR log below before changing behaviour — each ADR records *why* a decision was made and a
**Lens** for applying it to the next decision. This skill follows the docs contract in
[`.claude/rules/claude_skills/docs.md`](../../rules/claude_skills/docs.md) and the script
contract in [`.claude/rules/claude_skills/scripts.md`](../../rules/claude_skills/scripts.md).

## The development contract

Two commands are the entire loop for the gated code. Run from the repo root (never `cd` in):

```bash
make -C .claude/skills/mingrammer_diagrams/scripts fix   # auto-format + lint-fix (MUTATES source)
make -C .claude/skills/mingrammer_diagrams/scripts ci    # the gate: format-check, lint, mypy --strict, ≥90% cov, docs
```

`ci` must be 0-exit before handing work off. Use per-language sub-targets (`format-check-py`,
`lint-py`, `typecheck-py`, `test-cov-py`) only to narrow a failure; the contract is `ci`.

A third target, `diagrams`, re-renders the example PNGs and is **deliberately not part of
`ci`** (it needs Graphviz). Run it by hand after editing any `resources/examples/*.py`, then
`make docs` to refresh the gallery.

## File map

| File | Role |
|------|------|
| `SKILL.md` | Agent operating manual: layout engines, graph attrs, patterns, node-class references. |
| `README.md` | Human-facing consumer docs + architecture diagrams. |
| `scripts/_update_examples_readme.py` | **Private** helper: builds `resources/examples/README.md` from `.py`+`.png` pairs. |
| `scripts/test__update_examples_readme.py` | Pytest entry point (PEP-723), real-tmpdir tests, no mocks. |
| `scripts/conftest.py` | `importlib.reload` fixture so module-level lines count toward coverage. |
| `scripts/Makefile` | `fix` / `ci` / `docs` / `diagrams` and per-language sub-targets. |
| `resources/examples/*.py` | **Content**: self-contained, runnable diagram sources (the gallery's source of truth). |
| `resources/examples/*.png` | Committed renders of the matching `.py`. |
| `resources/examples/README.md` | Autogenerated gallery — never hand-edit. |
| `resources/*_nodes.md` | Provider symbol references (read for exact class names). |

## Architecture principles

- **Two kinds of Python, two regimes.** `resources/examples/*.py` are *content* (runnable
  diagram sources, exempt from the coverage gate — like `.mmd` fixtures). `scripts/*.py` are
  *gated code* (format/lint/mypy/≥90% cov). Keep that line sharp: a new example goes in
  `resources/examples/`, a new maintenance tool goes in `scripts/` with a test.
- **The gallery is a projection of the directory.** Adding a `.py`+`.png` pair and running
  `make docs` is the only way to change `examples/README.md`.
- **Examples self-render.** Each renders to `Path(__file__).with_suffix("")` so `uv run`-ing
  it writes the PNG beside the source, and `make diagrams` is a trivial loop.
- **`dot` + LR + ortho is the house style** (see SKILL.md); layout variants exist only as
  explicit, labelled examples.

## ADR log

### ADR-001 — Renamed AWS-only → multi-cloud `mingrammer_diagrams`
**Status:** Accepted.
**Context:** The skill was `aws_diagrams_mingrammer` with AWS-only examples, but the
underlying `diagrams` library renders GCP, Azure, Kubernetes and on-prem from the same
primitives. The AWS-only framing under-sold it, and GCP/Azure examples were requested.
**Decision:** Rename to `mingrammer_diagrams`; broaden the description, SKILL.md, and the
example gallery to AWS + GCP + Azure (same 3-tier shape per provider).
**Consequences:** The name now matches the library's reach; the gallery proves the layout
conventions are provider-agnostic. Old name archived under `tmp/archived/`.
**Lens:** Name a skill after the *capability* (the tool), not the first provider you happened
to demo. If a library spans N providers, the skill should too — add an example, not a caveat.

### ADR-002 — Example diagrams are content, not gated scripts
**Status:** Accepted.
**Context:** `scripts.md` requires every `scripts/*.py` to carry a test at ≥90% coverage. A
`diagrams` script is a thin Graphviz-rendering shell producing a binary PNG — there is no
meaningful pure logic to unit-test, and it hard-depends on the `dot` binary.
**Decision:** Keep the diagram sources in `resources/examples/` as self-rendering content
(the mingrammer analog of `mermaidjs_diagrams`' `.mmd` files). The only gated Python is the
private gallery generator.
**Consequences:** Examples stay copy-paste-runnable and the gate stays honest (it covers real
logic, not a renderer). Pyright flags the `>>` DSL and unused-expression statements in the
examples — expected and ignored, because they are not in the mypy gate.
**Lens:** Before forcing a coverage gate onto a file, ask "is this *logic* or *content*?"
Content (fixtures, runnable examples, data) belongs in `resources/`, exempt from coverage;
only logic goes in `scripts/` with a test. Mirror a sibling skill's proven data/code split.

### ADR-003 — The gallery README is generated and gated on `docs`
**Status:** Accepted.
**Context:** A hand-maintained gallery drifts the moment someone adds an example and forgets
to update the index.
**Decision:** `_update_examples_readme.py` builds `examples/README.md` from the `.py`+`.png`
pairs (alphabetical, `<!--TOC-->` populated by `mdtoc`). `docs` runs it; `ci` depends on
`docs`, so a stale gallery fails the gate.
**Consequences:** The gallery is always a faithful projection of the directory; drift is a
gate failure, not a silent rot.
**Lens:** If documentation can be derived from source-of-truth files, generate it and wire the
generator into `ci` — never trust humans to hand-sync an index.

### ADR-004 — `docs` asserts every `.py` has a `.png`; render stays out of `ci`
**Status:** Accepted.
**Context:** Rendering needs Graphviz, which isn't guaranteed in every CI environment — but a
gallery that links a missing image is worse than a loud failure.
**Decision:** `build_readme()` raises if any example lacks its rendered `.png`. `ci` runs
`docs` (cheap, no Graphviz) and so enforces the invariant; the actual PNG render (`diagrams`)
is a separate manual target.
**Consequences:** CI catches "added a source, forgot to render" without needing Graphviz,
while heavyweight rendering stays opt-in. Aligns with the project's fail-loud rule — no
silent skip of a missing artifact.
**Lens:** Split a pipeline at the dependency boundary: gate the *invariant* with the cheap,
dependency-free check; keep the heavy, tool-dependent step manual. Never let a generator
emit a reference to an artifact it didn't confirm exists.

## Extension checklist

Before you finish a change here:

- [ ] New diagram → a self-rendering `resources/examples/<name>.py` (PEP-723, `show=False`,
      render to `Path(__file__).with_suffix("")`); **not** a `scripts/` file (ADR-002).
- [ ] Rendered it (`make … diagrams`) and regenerated the gallery (`make … docs`) (ADR-003).
- [ ] New maintenance tool → `scripts/`, prefixed `_` if private, with a test + a
      `conftest.py` `importlib.reload` line, added to the Makefile `PRIVATE_PY_SCRIPTS`.
- [ ] `make … fix` run, then `make … ci` green (≥90% cov, mypy --strict, docs).
- [ ] README architecture diagrams still pass `mermaid_contrast.ts` **and**
      `mermaid_complexity.ts` (exit 0).
- [ ] No brand specifics introduced (`.claude/rules/agnostic_examples.md`).
- [ ] Every touched doc ≤ 500 lines.

## Known gotchas

- **Pyright noise in examples is expected.** The `diagrams` DSL overloads `>>`/`<<` and uses
  bare expression statements; Pyright reports `reportOperatorIssue` / `reportUnusedExpression`.
  These files are content under `resources/`, not in the mypy gate — do not "fix" them.
- **`Diagram(filename=...)` writes relative to cwd.** A bare `filename="x"` drops `x.png` in
  the *current* directory (repo root if run from there). Always derive from `__file__` so the
  PNG lands beside its source.
- **Graphviz cleans up its intermediate `.gv`** after a successful render — if you see a
  stray dot-source file, the render errored; read the traceback rather than committing it.
- **mdtoc header levels differ by file.** The gallery uses `--header-levels 2` (one entry per
  example); the README uses `4`. Match the existing invocation in the Makefile / mdtoc call.
- **`uv run --no-project`** is mandatory in the Makefile — without it, uv discovers the root
  `pyproject.toml` and injects unrelated coverage/plugin config that breaks the isolated run.
