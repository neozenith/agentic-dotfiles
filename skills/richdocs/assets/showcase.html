Data visualisation

Charts wear the brand.

Muted — grey is the default, colour is the exception

If everything is coloured, nothing is emphasised. Push every series to grey and let one keep its hue; the eye finds it before conscious reading. The greys sit below 3:1 by design, so the accented series is direct-labelled.

Sequential — one hue, light to dark

Magnitude. Lightness strictly monotone, so the steps can be ranked. Never a rainbow.

Diverging & status

Polarity, and the red-green trap.

Two poles, a neutral grey midpoint — never a hue at zero, or "no change" starts to look like a value. The poles must survive colourblindness, which is why green ↔ red is structurally impossible here: its poles collapse to ΔE ~1–3 under deuteranopia and convey nothing.

The honest test — simulated deuteranopia

The same scales rendered through the Machado-2009 matrix. This is what roughly 8% of men actually see.

Status — reserved, and never colour alone

Status colours are never a chart series. If they were, a red bar and a red error would be the same thing. Each carries a label, because colour may never carry meaning by itself.

One system, two zoom levels.

The same architecture as an at-a-glance overview and a detailed, clustered view. Dense diagrams are where layout, edge routing, and the themed palette are actually stress-tested — a five-box flow proves nothing.

Detail — subgraph clusters

3D embeddings · UMAP point cloud

Vector chunks in space.

The 3D view is not only for colour spaces. Here 72 text chunks across six topics are embedded, then reduced to 3D with UMAP — the kind of view that makes a RAG corpus debuggable: a mis-embedded chunk lands in the wrong cloud. Drag to orbit; hover a point for its chunk. The encoding colours are data, constant across themes; only the canvas re-skins.

Topic is the known label; cluster is what EVoC discovered from the vectors alone. Where they disagree, the embedding disagrees with the taxonomy.

Local structure — n_neighbors = 5

Tight, well-separated pockets. Emphasises which chunks are near-duplicates of each other.

Global structure — n_neighbors = 30

The same data, same transform, one knob turned: the coarse continent map, showing how topics sit relative to each other.

Compare two chunks — cosine distance

Click any two points to read their cosine distance on the raw vectors (the true semantic distance, whichever projection you are looking at). Click a selected point again to drop it. Selected points wear a bright halo.

Geographic · deck.gl + MapLibre

A real dark basemap, no vendor key.

The map rides a vector dark basemap (CartoDB dark-matter via MapLibre GL) with deck.gl arcs on top — free and keyless, crisp at any zoom. Traffic arcs fan from the Melbourne hub to the other Australian capitals, coloured from the brand. Drag to pan, scroll to zoom, right-drag to tilt.

In-browser SQL · duckdb-wasm

Query the data where it lives.

The DESIGN axis is only as strong as the DATA feeding it. Here an analytical SQL engine — DuckDB, compiled to WebAssembly — runs entirely in the browser: no server, no vendor key, no upload. Edit the query and run it; the result rows drive a themed chart and a table. The same rows are exactly the shape a graph or a map would consume, so one SQL step feeds any block on this page.

DuckDB loads on first run (several MB, once).
Architecture — editable draw.io SVG

Real stencils, still editable.

Icons are the vendored draw.io stencil pack. Each SVG carries its own mxfile source in a content attribute, so it re-opens in diagrams.net as first-class AWS & GCP shapes — not a flattened picture. Download one and keep editing.
